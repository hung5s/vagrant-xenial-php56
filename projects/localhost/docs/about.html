About databases

This box does not include any database server. We recommend you to install the database server(s) on your host machine for these reasons:

  • Although the purpose of using boxed development environment is to create one environment for all developers as well as test, production environment, there is a low chance that DB server cause a problem. Usually it is problem with locale or data format
  • You can work with the database(s) without your box running
  • You can use one DB server for multiple databases. If your projects relate, it is easier to have one DB server with all related databases
  • If you need a DB client software, running database server inside the box require additional port forwarding and Vagrant need to fix port collision when you have many boxes running

Quick Start

See Sample project for quickly setup an empty web project and check the PHP Information. This does not require SVN access to Tagrem Corp SVN server.

You need SVN access to Tagrem Corp SVN server to do the following.

To initialize a new Indition project, run php /host/projects/deploy/run.php Project/install --path=/host/projects/new-project

To replicate an existing Indition site:

  • Make sure you have access to that site's Admin to donwload the source infromation (domain.json) from Server Information page
  • Create an empty project folder
  • Copy the source file into your project folder, rename it to source.json
  • run php /host/projects/deploy/run.php Project/install --path=/host/projects/new-project
  • run php /host/projects/deploy/run.php Project/nginx --path=/host/projects/new-project to generate Nginx config file and follow the guide to reconfigure the web server

For more information, see Indition base project section
